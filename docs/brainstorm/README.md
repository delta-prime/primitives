# Brainstorm Threads

Design exploration and decision logs. Each thread captures a specific design question, the options considered, and the resolution.

## Naming Convention

`YYYY-MM-DD-topic.md`

## Structure

Each brainstorm doc should follow:

```markdown
# Topic Title

**Date:** YYYY-MM-DD
**Status:** open | resolved | deferred
**Resolution:** (if resolved) brief summary

## Context

What problem or question prompted this brainstorm?

## Options Considered

### Option A: Name
- Description
- Pros
- Cons

### Option B: Name
...

## Decision

What was decided and why?

## Consequences

What follows from this decision?

## Open Questions

What remains unresolved?
```
